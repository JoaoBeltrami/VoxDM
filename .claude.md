# Convenções do VoxDM

## Código
- Python 3.12.x — usar `uv venv --python 3.12` se der problema
- async/await em tudo que toca I/O
- Type hints obrigatórios em todas as funções
- Logs: structlog, nunca print()
- Erros: nunca `except: pass` — sempre logar com contexto
- Config: `from config import settings`, nunca `os.getenv()`

## Decisões travadas (não questionar)
- LLM: Groq `llama-3.3-70b-versatile`
- STT: Faster-Whisper tiny na GPU
- TTS: Edge TTS + Kokoro como fallback local
- Vetorial: Qdrant Cloud
- Grafos: Neo4j AuraDB
- Embeddings: sentence-transformers `all-MiniLM-L6-v2`

## Armadilhas conhecidas
- `kokoro` — pacote certo é `kokoro`, não `kokoro-tts`
- Gemini — usar `gemini-2.0-flash`, não `gemini-1.5-pro` (descontinuado)
- CUDA não detecta — quase sempre é Python 3.14, resolver com `uv venv --python 3.12`
- Makefile quebra — indentação com TAB, nunca espaço
- Context Builder — Working Memory sempre primeiro (40%), nunca cortar

## Ao criar arquivo novo
Adicionar seção no VOXDM_LEARN.md com 4 campos:
1. O que faz (1 frase)
2. Por que essa escolha (não "por que existe" — por que *essa* decisão)
3. Exemplo (entrada → saída real, funciona se copiar)
4. Short 30s (texto que Beltrami lê em voz alta, sem jargão)
